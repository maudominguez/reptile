Las pruebas deben ponerse en la carpeta reptile/bin/Debug

No se permiten dos metodos con el mismo nombre dentro de una clase.
No se permiten dos metodos con el mismo nombre en el scope global.
No soporta overloading.
En una subclase, sí se puede volver a usar el nombre de un método que fue usado en la superclase.
Dentro de un método, si dentro de una expresión aparece una variable, ésta se busca primero en las variables locales y, si no se encuentra ahí, se busca en las variables de instancia de la clase donde está el método.

Los arreglos se consideran objetos.
